## 1. Контекст и цель

ЧМ 2026 — самый масштабный в истории: 48 команд, 12 групп, 104 матча в США, Канаде и Мексике. Группа друзей хочет соревноваться между собой: кто лучше угадывает результаты, кто точнее выстраивает сетку. Никаких лишних фич, никаких незнакомых людей — только свои.

**Ключевая ценность:** видеть живые результаты, сравнивать свои прогнозы с корешами, не платить за готовые сервисы.

---

## 2. Пользователи

Один тип пользователя: **участник группы** — человек, которого позвали по инвайт-ссылке. Никакой публичной регистрации нет. Все равны, есть только один выделенный роль — **Admin** (тот, кто запускает сайт), он управляет настройками и может вручную подтвердить/откатить результаты.

## 3. Фичи — что строим

### 3.1 Аутентификация
- Вход только по **инвайт-ссылке** (`/join?code=XYZ`) или прямой ссылке на сайт
- Выбор никнейма + аватар (emoji-пикер или загрузка картинки)
- Сессия хранится в localStorage/cookie — ничего сложного
- Пароли не нужны, это не банк

### 3.2 Прогнозы на матчи

**Что можно прогнозировать:**
- Счёт каждого матча (точный результат: 2:1, 0:0 и т.д.)
- Кто выйдет из группы (топ-2 + лучшие 3-и места)
- Победитель каждого раунда плей-офф (R32 → R16 → QF → SF → Final)
- Чемпион турнира + лучший бомбардир (бонусный прогноз до старта)

**Дедлайны:**
- Прогноз на матч группового этапа закрывается **за 15 минут до старта матча**
- Прогноз на плей-офф закрывается **за 15 минут до старта конкретного матча**
- После закрытия — прогноз не изменить, но он виден другим

### 3.3 Турнирная сетка (Bracket Builder)

- Интерактивная сетка на весь плей-офф (R32 → R16 → QF → SF → Final)
- Каждый заполняет свою сетку до начала плей-офф (28 июня, после завершения группового этапа)
- Сетка автоматически заполняется реальными командами по ходу турнира
- Рядом с реальной сеткой — предсказанная: видно, где ошибся
- Можно переключаться между сетками разных участников

### 3.4 Живые результаты

- Список матчей с текущим счётом (обновление каждые ~60 секунд во время игры)
- Статус матча: ещё не начался / идёт (+ минута) / завершён
- Группы: позиции, очки, голы — живые таблицы групп
- Плей-офф: визуальная сетка с реальными результатами

### 3.5 Таблица лидеров (Leaderboard)

Каждый видит рейтинг всех участников. Очки начисляются автоматически после фиксации результата матча.

**Система очков:**

| Событие | Очки |
|---|---|
| Угадал исход (победа/ничья/поражение) | **2 очка** |
| Угадал точный счёт | **+3 очка** (итого 5) |
| Угадал победителя группы | **3 очка** |
| Угадал обоих выходящих из группы | **5 очков** |
| Угадал победителя раунда плей-офф | **3 очка** |
| Угадал финалистов | **5 очков** |
| Угадал чемпиона | **10 очков** |
| Угадал лучшего бомбардира | **5 очков** |

Лидерборд обновляется в реальном времени. После каждого матча участники видят, кто поднялся/упал.

### 3.6 Личный кабинет

- Все свои прогнозы + статус (ожидает / верно / неверно / частично)
- Точность в % (угаданные исходы / всего матчей)
- История: как менялся рейтинг по ходу турнира
- Сравнение с конкретным другом (кто из двух точнее)

## 4. Источник данных — API

### Главный вариант: `worldcup2026` (GitHub, open-source)

**URL:** `https://github.com/rezarahiminia/worldcup2026`  
**Базовый URL API:** `https://worldcup2026-api.up.railway.app` (или self-hosted)

✅ Полностью бесплатный  
✅ Не нужен API-ключ для read-доступа  
✅ Live scores, standings, матчи, команды, стадионы  
✅ Open source — можно задеплоить свою копию  

**Ключевые эндпоинты:**
```
GET /matches              — все 104 матча
GET /matches/live         — текущие матчи
GET /matches/{id}         — детали матча
GET /groups               — групповая стадия
GET /standings            — таблицы групп
GET /teams                — все 48 команд
```

### Резервный вариант: API-Football (api-sports.io)

**URL:** `https://v3.football.api-sports.io`  
**Параметры ЧМ 2026:** `league=1&season=2026`  
**Free tier:** 100 запросов/день — хватит для опроса раз в минуту  
⚠️ Нужна бесплатная регистрация для API-ключа

### Стратегия опроса

```
Матч не начался    → polling каждые 5 минут (статус расписания)
Матч идёт          → polling каждые 60 секунд (live score)
Матч завершён      → однократный финальный pull, дальше не опрашиваем
Нет живых матчей   → polling каждые 10 минут
```

## 5. Модель данных

### Таблицы (PostgreSQL / SQLite)

```sql
-- Пользователи
users (
  id, nickname, avatar, invite_code, is_admin,
  created_at, total_points
)

-- Матчи (синхронизируются из API)
matches (
  id, external_id, home_team, away_team,
  home_score, away_score, status,     -- 'scheduled' | 'live' | 'finished'
  minute, stage, group, match_date,
  venue, round
)

-- Прогнозы пользователей
predictions (
  id, user_id, match_id,
  predicted_home, predicted_away,
  points_earned, is_locked,           -- true = дедлайн прошёл
  created_at, updated_at
)

-- Прогнозы на сетку (bracket)
bracket_picks (
  id, user_id, round,                 -- 'R32' | 'R16' | 'QF' | 'SF' | 'Final'
  match_position, predicted_team,
  points_earned
)

-- Бонусные прогнозы
bonus_predictions (
  id, user_id, type,                  -- 'champion' | 'top_scorer'
  value, points_earned
)
```

## 6. Технический стек — рекомендации

Это для себя, деплой должен быть простым и дешёвым.

**Frontend:** Next.js (App Router) + Tailwind CSS  
**Backend:** Next.js API Routes (всё в одном репо) или Fastify  
**База данных:** PostgreSQL на Supabase (free tier) или SQLite если совсем просто  
**Деплой:** Vercel (frontend + API) — бесплатно для небольшого трафика  
**Фоновые задачи:** Vercel Cron Jobs для polling API каждые 60 сек  
**Реалтайм:** Supabase Realtime или Server-Sent Events для live обновлений  

### Структура репозитория
```
/app
  /api
    /matches        — прокси к внешнему API + кэш
    /predictions    — CRUD прогнозов
    /leaderboard    — расчёт очков
    /sync           — cron job для синхронизации матчей
  /matches          — страница матчей
  /bracket          — турнирная сетка
  /leaderboard      — таблица лидеров
  /profile/[user]   — профиль участника
/lib
  /scoring.ts       — логика начисления очков
  /api-client.ts    — клиент к worldcup2026 API
```

## 7. UX — ключевые экраны

### Главная (`/`)
- Матчи сегодня с текущим счётом
- Топ-3 лидерборда
- Кнопка "Сделать прогноз" на ближайший матч

### Матчи (`/matches`)
- Переключатели: Сегодня / Группа / Раунд / Все
- Карточка матча: флаги, счёт, статус, твой прогноз (после дедлайна — прогнозы всех)

### Прогноз (`/predict/[matchId]`)
- Выбор счёта скроллером (0–9 для каждой команды)
- Отображение прогнозов друзей (только после дедлайна)
- Таймер до закрытия ставки

### Сетка (`/bracket`)
- Левая половина: реальная сетка (обновляется из API)
- Правая половина / переключатель: моя сетка / сетка друга
- Подсветка верных/неверных прогнозов

### Лидерборд (`/leaderboard`)
- Таблица: место, аватар, никнейм, очки, %, матчей сыграно
- Фильтр по этапу (группы / плей-офф / всё)
- Анимация при обновлении (кто поднялся/упал)

### Профиль (`/profile/[userId]`)
- Статистика, история очков, прогрессия по турниру
- Список всех прогнозов с результатами

## 10. Открытые вопросы

1. **Счёт в дополнительное время**: считаем прогноз по итогу 90 мин или с учётом овертайма/пенальти?  
   → Рекомендация: прогноз только по 90 мин, отдельно угадать "кто пройдёт дальше" для плей-офф.

2. **Что если API ляжет?**: нужен ли ручной ввод результатов для Admin?  
   → Рекомендация: да, Admin-панель с кнопкой "Ввести результат вручную" как fallback.

3. **Инвайт-система**: просто ссылка или нужен лист одобренных никнеймов?  
   → Рекомендация: ссылка с кодом + Admin одобряет нового участника (одна кнопка).

4. **Очки за bracket**: считать баллы за каждый угаданный матч в сетке или только за финального чемпиона?  
   → Рекомендация: за каждый раунд, с нарастающим множителем (×1 R32, ×2 R16, ×3 QF, ×4 SF, ×5 Final).
